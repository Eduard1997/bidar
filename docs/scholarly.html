Architecture

High level overview

The web application is made up of multiple services that communicate with one another in order to provide the functionality expected by the user. These services are:

  1. Authentication service
  2. Main service
  3. Recommandations service
  4. Data refresh service
  5. User Profile Service

Front-end

Services

Authentication service

This service will check the user's credentials in order to make sure he is an already existing customer of the platform. From a technical point of view, this service will work using JWT token. The front-end, will make, on behalf of the user a 'log-in' request to this service, and the said service will check the credentials, and if they are fine(that is, this user is already present in the database) then it will generate a JWT token for the user and send it back to him in the response's payload. This JWT token will keep, besides an validity timeframe, metadata about the user like: the geographical location, its name etc and the preferences it has selected in the frontend UI.

Main service

Overview

This service, through the endpoints it will expose, will provide the main functionality of the platform. The endpoints will provide a REST interface that the front-end component can interact with through HTTP calls in order to allow the user to interact with the web application. Furthermore, all the requests that will be made to this service will be checked for the JWT token that is mandatory. In case the token isn't present, the request will be rejected.

Functionality

One of the features that will be provided by this service is the ability of retrieving, from the database, all the information of interest for the user based on it's metadata and interests.

Data refresh service

This service will help keep the data as fresh as possible. Basically, this service will have a scheduled job that runes every week which will download the latest version of the data set, parse it and update the database. Because this service doesn't expose any endpoints, it can't be accessed from the outside world, so no validation is needed.

User Profile Service

This service will expose a set of endpoints that will be called by the front end in order to allow the user to manage its profile and data. The endpoints exposed by this service, which will be called by the front end application, the user will be able to:

  1. Change its personal data, like: first name, last name, location
  2. Add, remove or update its area of interest
  3. Add, remove or update its area of skills
As with the other services, each request will be validated by checking that the JWT token is present in it.

Recommandations service

This service will expose a single endpoint, that will receive as input the user's pereferences and other input data provided by him and will sent back a list of recommended data and genres of intereset through the use of certain, special algorithms. This endpoint will be called by the front-end application when it has to display a page with the content of interest for the user. It goes without saying, that any request made to the endpoint exposed by the service will be valided in order to make sure the user is authenticated.
